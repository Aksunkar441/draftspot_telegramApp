# 📂 Шаг 2. Разбор файлов проекта «по полочкам»

Давай заглянем внутрь папок проекта. Тут нет ничего страшного — файлы разложены по логическим коробкам, прямо как детальки LEGO в разных контейнерах.

---

## 📦 Коробка 1: `backend/` (Серверная часть)

Здесь живет наш **Умный Судья** и **Почтовый Голубь**.

*   `requirements.txt` — это **список покупок**. Тут написано, какие сторонние библиотеки на Python нужно установить (например, `fastapi` для сервера, `aiogram` для бота, `sqlalchemy` для работы с базой данных).
*   `.env` — это **секретный листок**. Здесь хранятся пароли к базе данных и секретный токен твоего Telegram-бота. Его нельзя показывать другим!
*   `sql/schema.sql` — **чертеж базы данных**. Инструкции на языке SQL, которые объясняют базе данных, какие таблицы нужно создать.

### Внутри папки `backend/app/` (Сердце сервера):
*   `main.py` — **Главный Рубильник**. Когда мы запускаем сервер, компьютер первым делом читает этот файл. Он запускает веб-сервер и включает бота.
*   `config.py` — **Считыватель Секретов**. Он берет данные из `.env` и делает их доступными в коде.
*   `database.py` — **Трубопровод к Базе**. Он открывает соединение с Supabase, чтобы сервер мог читать и записывать данные.
*   **Папка `models/` (Модели):** Это пластилиновые формочки для базы данных. Здесь на языке Python написано, из каких полей состоят наши пользователи (`user.py`), площадки (`venue.py`) и матчи (`event.py`).
*   **Папка `api/` (Почтовые ящики):**
    *   `routes/users.py`, `events.py` и т.д. — это эндпоинты (адреса). Когда Фронтенд хочет что-то узнать, он отправляет запрос на эти адреса. Например, запрос к `/api/venues` вернет список всех полей.
    *   `deps.py` — **Контролер на входе**. Он проверяет, действительно ли запрос пришел из Telegram, и узнает, кто именно его отправил.
*   **Папка `bot/` (Логика Почтового Голубя):**
    *   `bot_instance.py` — создание самого бота.
    *   `handlers/start.py` — что бот делает при команде `/start`.
    *   `handlers/registration.py` — как бот по шагам спрашивает имя, возраст и сохраняет фото.
    *   `handlers/captain_actions.py` — обрабатывает нажатия кнопок «Принять» или «Отклонить», которые капитан видит прямо в чате Telegram.
*   **Папка `services/` (Умные Службы):**
    *   `event_service.py` — содержит логику «принятия заявки» с защитой от переполнения мест.
    *   `notification_service.py` — помогает отправлять сообщения пользователям через бота.
    *   `reminder_service.py` — **Будильник**. Он раз в минуту проверяет, нет ли игр, которые начнутся через 2 часа, чтобы напомнить игрокам.

## 📦 Коробка 2: `frontend/` (Визуальная часть)

Здесь живет то, что видит пользователь на экране. Она написана на **Vue.js 3** с использованием инструмента сборки **Vite**.

*   `package.json` — это **список покупок для фронтенда**. Содержит список JavaScript-библиотек.
*   `vite.config.js` — **настройка сборщика**. Мы настроили его так, чтобы он работал как мост и автоматически передавал запросы на наш Бэкенд.
*   `index.html` — **главная рамка сайта**. Простой HTML-файл, в который встраивается наше приложение.

### Внутри папки `frontend/src/` (Код интерфейса):
*   `main.js` — **Стартовый файлик**. Запускает Vue и подключает стили и плагины.
*   `App.vue` — **Главный экран**. В него вставляются все остальные страницы.
*   **Папка `views/` (Страницы сайта):**
    *   `FeedView.vue` — **Лента матчей**. Здесь показываются карточки игр. Их можно пролистывать.
    *   `CreateEventView.vue` — **Страница создания игры**. Форма, где выбирают вид спорта, площадку, время и цену.
    *   `EventDetailView.vue` — **Карточка игры изнутри**. Тут видно список участников и кнопку «Вступить». Если зашел капитан этой игры, он также видит список желающих вступить и кнопки принять/отклонить.
    *   `HistoryView.vue` — **Мои игры**. Поделена на три вкладки: игры, которые создал я; игры, куда я подал заявку; и игры, куда меня уже приняли.
*   **Папка `stores/` (Локальная Память Сайта):**
    *   `profile.js` — хранит информацию о текущем вошедшем пользователе.
    *   `events.js` — хранит список матчей, чтобы не скачивать их заново при каждом действии.
*   **Папка `api/` (Телефонный кабель):**
    *   `client.js` — специальный клиент Axios. При каждом запросе к бэкенду он автоматически добавляет подпись авторизации Telegram.
